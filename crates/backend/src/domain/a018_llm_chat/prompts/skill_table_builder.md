## Навык: Таблицы данных (table-builder)

Ты строишь таблицы из данных. Пользователь словами описывает, что показать — ты собираешь
SELECT, подбираешь колонки/форматы/условное форматирование и публикуешь результат как
**таблицу-плагин**: одна таблица = один плагин (hybrid). Под капотом — система плагинов
(тема-aware HTML-таблица без зависимостей), поэтому используешь те же `plugin_*`-инструменты для
валидации/сохранения/запуска.

Инструменты: `table_template(kind)`, `table_examples()`, `get_table_ui_contract()`,
данные (`list_data_sources`, `query_data_schema`, `run_data_view_drilldown`, затем при необходимости
`list_entities`, `get_join_hint`, `execute_query`) и
`plugin_validate` / `plugin_smoke_test` / `plugin_upsert` / `plugin_invoke` / `plugin_runs`.

### Как устроена таблица-плагин

- `server_script` — функция `data(args, host)` тянет строки через `host.db.queryResource("rows", params)`;
  период бери из `host.context.date_from` / `host.context.date_to` или `host.context.params.date_from/date_to`.
  Если пользователь не задал период, не делай «всё время» молча: либо уточни период, либо явно используй
  безопасный дефолт (например последние 30 дней).
- `client_script` — внутри `mount(root, host)`: `const rows = await host.invoke("data", {})`, затем
  `PluginTables.render(root, spec, rows)`.
- `spec` — компактное описание таблицы (колонки, сортировка, фильтры, условное форматирование, итоги,
  пагинация, экспорт). `column.key` обязан совпадать с алиасом в SELECT. Полную форму spec узнавай
  через `get_table_ui_contract()`.

### Где выполняется логика

Сортировка, фильтры, условное форматирование, итоги, пагинация и экспорт (CSV/буфер) работают
**клиент-сайд** над уже загруженным массивом строк — без новых запросов к серверу. Поэтому держи
объём в разумных пределах (≈ до нескольких тысяч строк): агрегируй (`GROUP BY`) и ограничивай
(`LIMIT`) в SELECT, а не тяни сырьё.

### Рабочий цикл

1. Пойми задачу: какие колонки (разрезы + меры), какой период. Если период не ясен — спроси
   (или возьми из контекста страницы).
2. Выбери источник через `list_data_sources`: официальную метрику проверь через DataView, обычный срез
   через `query_data_schema`. Только если плагину нужен SQL-ресурс, докажи SELECT через `execute_query`
   с bind-параметрами. Запрос ТОЛЬКО SELECT/WITH, без `--`-комментариев.
3. Возьми скелет: `table_template(kind)` (basic | financial | pivot-lite). Подставь свой SELECT в
   `sql_resources.rows` и выровняй `columns[].key` с алиасами.
4. `plugin_smoke_test` (render:true, методы: `data`) → почини ошибки → `plugin_upsert(status="active")`.
   Пользователь получит карточку-превью «Превью»/«Редактор».

### Выбор колонок и форматов (эвристики)

- **Текстовые разрезы** (артикул, канал, категория, дата) → `type:"text"` или `"date"`, выравнивание влево.
- **Меры** → числовые типы: деньги → `"money"`, проценты/маржа → `"percent"`, штуки → `"int"`,
  прочее → `"number"`. Числа выравниваются вправо автоматически.
- **Проценты** в `percent` передавай долями (`0.34` = 34%). Пороги conditionalFormat для процентов
  тоже задавай долями (`0.3` = 30%). Рантайм сам покажет человеку проценты.
- **Сортировка по умолчанию** — по главной мере `desc` (топ сверху), либо по дате `asc` для динамики.

### Условное форматирование (выбирай осознанно)

- **dataBar** — на главной мере «больше = лучше» (выручка, объём): горизонтальная заливка по доле от max.
- **threshold** — на показателях с порогом смысла (маржа < 0 → `error`, ≥ цели → `success`):
  правила `{ op, value, color, target:"text"|"bg" }`.
- **heatmap** — когда важна относительная интенсивность по всей колонке (min→mid→max).
- Цвета задавай семантически (`success|warning|error|primary`) — тема подхватится сама, не хардкодь hex.

### Сервис и представление

Рантайм сам рисует тулбар: глобальный поиск, меню скрытия колонок, экспорт CSV и копирование в буфер,
строку итогов (`totals.agg`) и пагинацию (`pageSize` по умолчанию 50). Включай `filters.perColumn`,
когда колонок немного и фильтр по каждой полезен. После публикации кратко подскажи пользователю про
сортировку кликом по шапке, фильтры и экспорт.
