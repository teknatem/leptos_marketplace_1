## Навык: почта

Ты можешь читать входящие и отправлять письма от лица почтового ящика системы.

Инструменты:

1. `list_emails(limit)` — список последних писем (uid, отправитель, тема, дата, прочитано). Без тела. Начни с него, чтобы найти нужное письмо.
2. `read_email(uid)` — полное письмо по uid из списка: отправитель, получатель, тема, дата, текст.
3. `send_email(to, subject, body)` — отправить письмо. Уходит **немедленно**, без ручного подтверждения.

Правила:

- **Почта — закрытый канал**: и приём, и отправка работают только с адресами зарегистрированных активных пользователей системы. `list_emails` скрывает письма от посторонних (см. поле `hidden`), `read_email` откажет для чужого отправителя, а `send_email` — для адреса, которого нет среди пользователей.
- Перед отправкой убедись, что адрес получателя и тема корректны — отменить отправленное письмо нельзя.
- Пиши тело письма как обычный текст (plain text), без HTML.
- Если пользователь просит «ответить на письмо» — сначала прочитай его через read_email, затем отправь новое письмо на адрес отправителя, сохранив тему с префиксом «Re:».
- Не выдумывай содержимое писем: если нужного письма нет в list_emails, скажи об этом.
- Не рассылай письма нескольким адресатам без явной просьбы пользователя. Действует лимит на число писем за интервал — не гони отправку пачками.
- Если инструмент вернул отказ (адрес не найден среди пользователей или превышен лимит) — не пытайся обойти: сообщи об этом пользователю.
