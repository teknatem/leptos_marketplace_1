Не превышать уровень вложенности 2–3 в одном фрагменте кода.

Использовать ранний выход из функции (return, ? для обработки ошибок) вместо глубоких вложенных условий.

Вынести вложенную логику в отдельные функции, чтобы уменьшить вложенность в основной функции.

Применять конструкции типа match и комбинаторы (map, and_then) для замены вложенных if-else.

Избегать вложенных циклов и условий внутри друг друга, если возможна их замена на последовательные шаги.

Использовать логические операторы (&&, ||) для объединения условий вместо вложенных if.

Следить за читаемостью: если код становится трудно воспринимаемым из-за вложений, проводить рефакторинг.
