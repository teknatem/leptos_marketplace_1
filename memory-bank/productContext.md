# Product Context

## Почему проект существует

Leptos Marketplace нужен как **единое self-hosted веб-приложение для локального запуска** для компаний, которые ведут учёт в **1С:УТ11** и работают с маркетплейсами (Wildberries, Ozon, Яндекс.Маркет и др.). Цель — убрать ручные сверки и разрозненные источники данных, дать единый интерфейс для управления подключениями, справочниками и импортами.

## Какие проблемы решаем

- Синхронизация справочников и данных между 1С и маркетплейсами без ручных операций
- Контроль подключений и параметров интеграций (токены/ключи/режимы)
- Удобный просмотр и обработка больших списков документов (заказы/продажи/транзакции)
- Сводная аналитика и отчётность по нескольким каналам продаж

## Как продукт должен работать (UX goals)

- **Быстро и предсказуемо**: списки и формы открываются мгновенно, понятные статусы загрузки/ошибок
- **Единые паттерны UI**: одинаковая структура списков, форм и модалок по всему приложению
- **Минимум кликов**: основные действия (создать/обновить/удалить/тест) всегда рядом и очевидны
- **Стабильная self-hosted работа**: локальная SQLite, без внешней инфраструктуры, запуск через браузер
- **Безопасность**: multi-user и JWT-авторизация для системных разделов

## Ключевые интерфейсные соглашения

- UI строится на **Leptos 0.8 + Thaw UI**
- В проекте есть 2 типа таблиц:
  - **SimpleTable**: Thaw-таблица, fetch-all, клиентская сортировка, без пагинации (для малых справочников/настроек)
  - **ComplexTable**: списки по List-Standard с серверной пагинацией offset/limit, фильтрами, итогами и т.п.
